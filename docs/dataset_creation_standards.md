# 多模态审核模型：旗帜、地图与敏感人物数据集制作标准

为确保大模型（VLM）及计算机视觉模型（CV）能够精准学习、识别违规旗帜与问题地图，提升审核准确率并降低误判，特制定本数据集采集与加工制作标准。所有训练集及评测集素材必须严格遵循以下规范。

## 一、 旗帜类数据制作标准

### 1. 图像采集与裁切标准
- **核心原则**：**拒绝原始视频全屏截图！** 绝不允许将包含大量无关背景、人物、字幕的整帧视频画面直接作为训练集。
- **精细化裁切 (Crop)**：所有旗帜图片必须经过人工裁切，确保画面主体 100% 聚焦于旗帜本身。必须剔除任何可能对模型造成干扰的杂乱背景（如街道、人群、房间内饰等）。
- **清晰度要求**：裁切后的旗帜图片必须清晰易辨认，模糊到无法看清核心图案的废片一律废弃。

### 2. 数据多样性与数量要求
每种违规旗帜必须包含丰富的物理形态与视角变化：
- **数量下限**：每种旗帜**不少于 10 张**独立图片。**旗帜类总数据量要求不低于 1000 张**。
- **角度覆盖**：必须涵盖正面平铺、侧面透视、仰视、俯视等不同角度。**每个主要角度不少于 2 张**。
- **形态覆盖**：必须包含旗帜在真实场景中的各种物理形态，包括但不限于：
  - 扭曲/卷曲遮挡状态
  - 随风飘荡/飘扬状态
  - 垂落/褶皱状态
  - 不同的光照条件（暗光、强光、背光）

### 3. 文本描述 (Captioning) 标准
仅有图片不足以构成高质量数据集。每张独立裁切的旗帜图片**必须**配备详细的对齐文本描述，引导模型理解其核心特征。
- **描述要求**：必须准确描述旗帜的底色、主要图形元素、特殊文字或符号及其相对位置，**并附带该旗帜的违规含义描述**。
- **示例**：
  - *“这是一面雪山狮子旗。背景由红蓝相间的放射状条纹组成。中央下方有一座白色的雪山，雪山前有两只绿鬃白狮相对而立，共同托举着一个燃烧的三色太极图形，雪山上方有一轮黄色的太阳。该旗帜是‘西藏独立运动’的标志，属于严重违规的涉政分裂内容。”*

### 4. 正误示例对比

## 二、 地图类数据制作标准

地图类数据的审核极度敏感，主要涉及政治合规性问题。其数据制作标准在清晰度与裁切要求上与旗帜类一致，但在**分类与描述**上有更严格的政审标准。

### 1. 图像采集与裁切标准
- 地图图像必须足够清晰，能够辨认出国界线、岛屿点位及重要地名。
- 严禁包含无关的主持人、复杂的演播室背景。必须精准裁切出地图图块区域。
- **数量下限**：每种违规地图**不少于 10 张**独立图片（需包含不同形态和角度）。**地图类总数据量要求不低于 1000 张**。

### 2. 重点政治问题分类及特征标注
每张问题地图必须明确指出其违规的**具体政治原因**，并在文本描述（Caption）中详细写明。
常见的核心问题包括但不限于：
1. **未包含台湾岛**：中国地图中将台湾岛遗漏或错误涂色（如与其他国家同色）。
2. **漏绘南海诸岛及九段线/十段线**：中国全图缺乏完整的南海区域及国界线标示。
3. **藏南地区与阿克赛钦地区国界线错误**：将中国领土错误划给邻国。
4. **钓鱼岛及其附属岛屿漏绘或错名**。

### 3. 文本描述 (Captioning) 标准
- **描述要求**：不仅要描述这是一张地图，还必须“指出错在哪里”。
- **示例**：
  - *“这是一张存在政治问题的中国地图。图片中仅绘制了中国大陆部分，未包含台湾岛，且完全漏绘了南海九段线及南海诸岛，属于严重违规地图。”*

### 4. 正误示例对比

**【错误示例 ❌】**
**漏绘台湾岛及九段线，属于常见的严重涉政违规地图。**

**【正确示例 ✅】**
**包含完整的台湾岛及南海九段线，符合国家标准。**

## 三、 人物类（敏感/劣迹人物）数据制作标准

人物类（尤其是劣迹艺人、敏感政治人物）的面部特征识别是审核的核心之一。此类数据需满足极高的纯净度，避免引入干扰特征。

### 1. 图像采集与裁切标准
- **取景范围限制**：要求**只有头部，或半身相**。全身照或远景照通常面部像素过低，严禁直接作为训练集，必须进行裁剪放大。
- **人数限制**：**一张图片只能包含一个人**，严禁出现多个人物同框（合照、人群背景等必须裁切或剔除），防止模型特征混淆。
- **面部清晰度要求**：**人脸必须清晰、无遮挡**。严禁使用戴墨镜、戴口罩、面部被话筒或大号字幕遮挡的图片。
- **数量下限**：每个人物**至少 5 张**独立图片，必须包括正脸和侧脸以及不同角度。**人物类总数据量要求不低于 5000 张**。

### 2. 正误示例对比

**【错误示例 ❌】**
**错误特征**：多人合照、包含全身远景、面部被口罩或麦克风遮挡、画质模糊，直接视频截图。

## 四、 数据输出格式与交付标准

在完成图片的搜集、裁切与描述编写后，需将结构化数据进行汇总并随图片统一交付。

### 1. 文件格式与列定义
- **格式要求**：需输出为 `.csv` 或 `.xlsx` (Excel) 表格文件。
- **字段要求**：
  - **第 1 列 (`image_filename`)**：图片对应的完整文件名（必须包含扩展名，如 `.jpg` / `.png`）。
  - **第 2 列 (`description`)**：图片的详细特征描述与违规含义说明。
  - **第 3 列 (`person_name`)**：**仅当该数据分类为“人物（敏感/劣迹人物）”时必须包含此列**，用于填写图片中人物的真实姓名。旗帜或地图类数据本列可留空或不包含此列。

### 2. 交付目录结构示例
交付时，需将图片文件夹与表格文件放置在同一层级：
```text
data_example/
├── metadata.csv
└── images/
    ├── flag_001.jpg
    ├── map_001.png
    └── person_001.jpg
```

### 3. 表格内容示例
| image_filename | description | person_name |
| :--- | :--- | :--- |
| flag_001.jpg | 这是一面雪山狮子旗...属于涉政违规内容。 | |
| map_001.png | 中国地图缺失台湾岛及九段线，属于严重问题地图。 | |
| person_001.jpg | 该人物面部五官清晰，单人半身照，无遮挡物。 | 张三 |
